# Code of Conduct

We expect respectful, professional collaboration across product, engineering, security, and domain contributors.

## Expected Behavior

- Be precise, constructive, and kind in reviews.
- Respect privacy, security, and compliance boundaries.
- Report vulnerabilities through the security process instead of public comments.

## Unacceptable Behavior

- Harassment, discrimination, or abusive conduct.
- Sharing secrets, protected data, or tenant data.
- Misrepresenting placeholder functionality as production capability.

Maintainers may remove comments, close issues, or restrict participation when needed to protect the project and its users.
